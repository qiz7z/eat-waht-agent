## 修复内容

### 1. 清理死代码（约 726 行）

**问题**：项目中存在 4 个未被使用的文件，增加了维护负担。

**修复**：
- 删除 `agent/context.py`（108 行）
- 删除 `agent/preference.py`（198 行）
- 删除 `agent/recommendation.py`（233 行）
- 删除 `agent/controller.py`（187 行）
- 删除相关测试文件：`test_context.py`、`test_preference.py`、`test_controller.py`
- 更新 `README.md` 移除对这些文件的引用

**效果**：减少了约 726 行无用代码，提高了代码库的可维护性。

### 2. 修复 timeout 参数传递

**问题**：`config.py` 中的 `build_chat_openai_kwargs` 函数没有包含 `timeout` 参数，导致 `LLM_TIMEOUT` 配置无法传递给 `ChatOpenAI`。

**修复**：
- 在 `build_chat_openai_kwargs` 函数中添加 `timeout` 参数
- 更新相关测试用例

**效果**：确保 LLM 调用能够正确使用配置的超时时间。

### 3. 添加日志脱敏功能

**问题**：`api.py` 中的日志记录了完整的用户输入，存在安全风险。

**修复**：
- 在 `api.py` 中添加 `_sanitize_log_message` 函数
- 对用户输入进行脱敏处理：
  - 截断过长的消息（默认 100 字符）
  - 移除邮箱地址
  - 移除手机号
  - 移除身份证号
- 修改日志记录调用，使用脱敏函数

**效果**：提高了日志安全性，避免泄露用户敏感信息。

### 4. 修复数据库路径为绝对路径

**问题**：`food_data_manager.py` 和 `crawler.py` 使用相对路径 `"data"`，部署时可能找不到数据目录。

**修复**：
- 修改 `data_dir` 参数默认值为 `None`
- 使用 `Path(__file__).parent.parent / "data"` 确保路径正确
- 同时修复 `food_data_manager.py` 和 `crawler.py`

**效果**：提高了部署可靠性，确保数据目录在任何位置都能正确访问。

### 5. 修复日志配置线程安全问题

**问题**：`logging_config.py` 中的 `_configured` 标志没有锁保护，多线程环境下可能重复初始化。

**修复**：
- 添加 `threading.Lock`
- 使用双重检查锁定模式（Double-Checked Locking）

**效果**：提高了多线程环境下的安全性。

## 已知遗留问题

以下问题在本次修复中未处理，但已添加注释说明：

1. **全局字典多 worker 不兼容**（`tools.py:23`）
   - 问题：`_agent_states` 是进程内全局字典，uvicorn 多 worker 模式下无法共享
   - 状态：已添加注释说明，当前单 worker 模式可接受
   - 建议：生产环境使用 Redis 等外部存储

2. **`__del__` 资源释放不可靠**（`agent.py:194`）
   - 问题：Python 的 `__del__` 不保证被调用，且调用顺序不可控
   - 状态：已添加注释说明
   - 建议：生产环境使用上下文管理器（`with` 语句）

3. **CORS 配置不当**（`api.py:30`）
   - 问题：默认 `allow_origins=["*"]`，生产环境应限制
   - 状态：可通过环境变量配置，开发环境可接受
   - 建议：生产环境配置具体的域名白名单

4. **`generate_recommendation` 工具硬编码**（`tools.py:182-233`）
   - 问题：5 种口味的推荐数据硬编码在代码中
   - 状态：与新的食物数据库系统重复，需要进一步整合
   - 建议：移除旧的推荐系统，统一使用食物数据库
